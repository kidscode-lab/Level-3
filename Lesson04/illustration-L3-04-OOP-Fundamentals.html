Classes & Objects

Understanding the building blocks of OOP

🎯 What You'll Learn

📋

Classes

Think of a class as a blueprint or template. Just like a cookie cutter makes cookies, a class makes objects!

🍪

Objects

Objects are actual "things" created from classes. Each object has its own unique data but follows the same pattern.

🔧

Attributes

These are the characteristics or properties that objects remember, like a pet's name or hunger level.

🏃

Constructor

The special __init__ method that sets up your object when it's first created, like naming your pet.

💡 Real-World Example

🚗 Think of a Car Blueprint

A car blueprint shows where the engine, wheels, and seats go, but it's not a real car. When manufacturers use that blueprint, they create actual cars. Each car has the same basic structure but different colors, interiors, and owners driving them.

Similarly, a Pet class is like a blueprint that defines what all pets should have (name, hunger, happiness), but each pet object is unique with its own values!

🔨 Basic Pet Class

Let's create our first pet class: 
class Pet:
    def __init__(self, name):
        # These are attributes - each pet remembers its own values
        self.name = name
        self.hunger = 5
        self.happiness = 5
    
    def say_hello(self):
        # This is a method - something the pet can do
        print(f"🐾 Hi! I'm {self.name}! Nice to meet you!")
    
    def show_status(self):
        # Show the pet's current condition
        print(f"📊 {self.name}'s Status:")
        print(f"   🍽️ Hunger: {self.hunger}/10")
        print(f"   😊 Happiness: {self.happiness}/10")

Methods & Behavior

Making your pets interactive and responsive

🎯 What You'll Learn

🔧

Methods

Functions that belong to objects. They define what your pet can do, like eat, play, or sleep.

📝

State Changes

Methods can change object attributes. When a pet eats, its hunger goes down!

🎮

Input Validation

Make sure your pet can't eat when it's not hungry or play when it's too tired.

💬

Feedback

Give users messages about what happened when they interact with their pet.

🍽️ Enhanced Pet with Actions

Let's add interactive behaviors to our Pet class: 
class Pet:
    def __init__(self, name):
        self.name = name
        self.hunger = 5
        self.happiness = 5
        self.energy = 5
    
    def say_hello(self):
        print(f"🐾 Hi! I'm {self.name}! Nice to meet you!")
    
    def feed(self):
        # Method that changes the pet's state
        if self.hunger > 0:
            self.hunger -= 1
            self.happiness += 1
            print(f"🍽️ {self.name} is eating! Yum yum!")
            print(f"   Hunger: {self.hunger}/10, Happiness: {self.happiness}/10")
        else:
            print(f"😋 {self.name} is already full! No more food needed.")
    
    def play(self):
        # Playing makes pet happy but costs energy
        if self.energy > 0:
            self.energy -= 1
            self.happiness += 2
            self.hunger += 1
            print(f"🎾 {self.name} is playing! So much fun!")
            print(f"   Energy: {self.energy}/10, Happiness: {self.happiness}/10")
        else:
            print(f"😴 {self.name} is too tired to play. Let them rest!")
    
    def sleep(self):
        # Sleeping restores energy
        self.energy += 3
        self.happiness += 1
        # Keep values in reasonable range
        if self.energy > 10:
            self.energy = 10
        if self.happiness > 10:
            self.happiness = 10
        print(f"💤 {self.name} takes a nice nap! Zzz...")
        print(f"   Energy: {self.energy}/10, Happiness: {self.happiness}/10")
    
    def show_status(self):
        print(f"📊 {self.name}'s Status:")
        print(f"   🍽️ Hunger: {self.hunger}/10")
        print(f"   😊 Happiness: {self.happiness}/10")
        print(f"   ⚡ Energy: {self.energy}/10")

🧠 Key Concepts

💡 Method Design Principles

  • Input Validation: Check if actions are possible before doing them
  • State Management: Keep attribute values within reasonable ranges
  • User Feedback: Always tell the user what happened
  • Logical Consequences: Actions should have realistic effects

🐾 Advanced Pet Class

Let's enhance our Pet class with more attributes and smarter behavior: 
class Pet:
    def __init__(self, name):
        self.name = name
        self.hunger = 5
        self.happiness = 5
        self.energy = 5
        self.health = 10
        self.cleanliness = 5
    
    def _keep_in_bounds(self, value, min_val=0, max_val=10):
        # Helper method to keep values in range
        return max(min_val, min(max_val, value))
    
    def get_mood(self):
        # Calculate overall mood based on all stats
        total_wellness = (
            self.health + 
            self.happiness + 
            self.energy + 
            (10 - self.hunger) + 
            self.cleanliness
        ) / 5
        
        if total_wellness >= 8:
            return "🌟 Excellent"
        elif total_wellness >= 6:
            return "😊 Happy"
        elif total_wellness >= 4:
            return "😐 Okay"
        elif total_wellness >= 2:
            return "😟 Worried"
        else:
            return "😢 Very Sad"
    
    def feed(self):
        if self.hunger > 0:
            self.hunger -= 2
            self.happiness += 1
            self.health += 1
            self.cleanliness -= 1  # Eating can be messy!
            
            # Keep all values in bounds
            self.hunger = self._keep_in_bounds(self.hunger)
            self.happiness = self._keep_in_bounds(self.happiness)
            self.health = self._keep_in_bounds(self.health)
            self.cleanliness = self._keep_in_bounds(self.cleanliness)
            
            print(f"🍽️ {self.name} enjoys a delicious meal!")
        else:
            print(f"😋 {self.name} is already full!")
    
    def play(self):
        if self.energy > 1:
            self.energy -= 2
            self.happiness += 3
            self.hunger += 1
            self.cleanliness -= 2  # Playing gets dirty!
            
            # Keep all values in bounds
            self.energy = self._keep_in_bounds(self.energy)
            self.happiness = self._keep_in_bounds(self.happiness)
            self.hunger = self._keep_in_bounds(self.hunger)
            self.cleanliness = self._keep_in_bounds(self.cleanliness)
            
            print(f"🎾 {self.name} has a blast playing!")
        else:
            print(f"😴 {self.name} is too tired to play!")
    
    def clean(self):
        self.cleanliness += 4
        self.happiness += 1
        self.health += 1
        
        self.cleanliness = self._keep_in_bounds(self.cleanliness)
        self.happiness = self._keep_in_bounds(self.happiness)
        self.health = self._keep_in_bounds(self.health)
        
        print(f"🛁 {self.name} gets a nice, refreshing bath!")
    
    def sleep(self):
        self.energy += 4
        self.happiness += 1
        self.hunger += 1
        
        self.energy = self._keep_in_bounds(self.energy)
        self.happiness = self._keep_in_bounds(self.happiness)
        self.hunger = self._keep_in_bounds(self.hunger)
        
        print(f"💤 {self.name} takes a peaceful nap!")
    
    def show_status(self):
        print(f"📊 {self.name}'s Complete Status:")
        print(f"   🍽️ Hunger: {self.hunger}/10")
        print(f"   😊 Happiness: {self.happiness}/10")
        print(f"   ⚡ Energy: {self.energy}/10")
        print(f"   ❤️ Health: {self.health}/10")
        print(f"   🧼 Cleanliness: {self.cleanliness}/10")
        print(f"   🎭 Mood: {self.get_mood()}")

⚠️ Common Mistakes to Avoid

  • Forgetting to keep values within bounds (0-10)
  • Making actions affect only one attribute when they should affect multiple
  • Not considering realistic consequences (playing makes you dirty and hungry)
  • Forgetting to give user feedback about what happened
